# Data Model: Contextual Retrieval, Breadcrumb Injection, Multi-Source Corpus & Per-Category Benchmarking

**Phase 1 output for feature 010**

---

## Entities

### 1. Breadcrumb

A structural location path extracted from the document's heading hierarchy and prepended to each chunk.

**Has no standalone existence** — it is embedded in the chunk's stored text and in a dedicated ChromaDB metadata field.

| Attribute | Type | Description |
|-----------|------|-------------|
| `breadcrumb` | `str` | Formatted heading path, e.g. `"Player's Guide > Chapter 2: Races > Dwarf"`. Empty string when no heading is available. |

**Format rule**: `"{doc_name}"` (document only) → `"{doc_name} > {H1}"` → `"{doc_name} > {H1} > {H2}"` → `"{doc_name} > {H1} > {H2} > {H3}"`. Deepest heading level available is always used (FR-005).

**Fallback rule**: When no heading precedes a chunk, the document filename stem is used as the sole breadcrumb component (FR-006).

**Storage**:
- Prepended to `original_text` in ChromaDB metadata: `"{breadcrumb}\n\n{raw_chunk_text}"`
- Also stored as a separate `breadcrumb` string metadata field in ChromaDB
- Populated into `KnowledgeChunk.breadcrumb` at retrieval time

---

### 2. Contextual Summary

A 1–2 sentence description generated by the LLM at ingestion time, situating the chunk within the document for the purpose of improving semantic retrieval.

**Has no standalone existence** — merged into the compound embedding text. Not stored separately in metadata.

| Attribute | Type | Description |
|-----------|------|-------------|
| Inline in compound text | `str` | One or two sentences. Prepended after the breadcrumb, before the LLM headline/summary. |

**Generation conditions** (FR-007, FR-008, FR-009):
- Only generated when `enable_contextual_summaries=True` is passed to `IngestionPipeline.run()`
- Failure falls back to no summary — chunk is ingested using breadcrumb + headline + enrichment.summary + raw text
- Each attempt is logged at INFO level; fallbacks at WARNING level

**Compound text order** (full enrichment path):
```
{breadcrumb}

{contextual_summary}   ← only when enable_contextual_summaries=True and generation succeeds

{headline}             ← from ChunkEnricher (existing)

{enrichment.summary}  ← from ChunkEnricher (existing)

{raw_chunk_text}
```

---

### 3. Source Type

A classification tag applied to all chunks produced from a given document at ingestion time.

| Attribute | Type | Allowed values |
|-----------|------|---------------|
| `source_type` | `str` | `"rulebook"`, `"supplement"`, `"handwritten_note"`, `"novel"` |

**Default**: `"rulebook"` (FR-011). Existing chunks in the index without this field are treated as `"rulebook"` via `.get("source_type", "rulebook")` fallback.

**Storage**: Stored as a string metadata field in ChromaDB on every chunk. Populated into `KnowledgeChunk.source_type` at retrieval time.

**No retrieval filter**: `source_type` is stored for context enrichment and future tooling; `KnowledgeRetriever.search()` does not accept a `source_type` filter. Source-type awareness belongs in the LLM/prompt layer.

---

### 4. Category Score

Per-question-category metric bundle computed during benchmark evaluation.

| Attribute | Type | Description |
|-----------|------|-------------|
| `mean_mrr` | `float` | Mean Reciprocal Rank for questions in this category |
| `mean_ndcg` | `float` | Mean nDCG@k for questions in this category |
| `mean_recall_at_k` | `float` | Mean Recall@k for questions in this category |
| `question_count` | `int` | Number of questions in this category |

**Categories** (from gold standard): `direct_fact`, `comparison`, `holistic`, `numeric`, `relationship`, `uncategorized` (catch-all for questions without a `category` field — FR-003).

## Schema Changes

### `IngestionConfig` (packages/rag/rag/knowledge/interface.py) — NEW

```python
@dataclass
class IngestionConfig:
    source_type: Literal["rulebook", "supplement", "handwritten_note", "novel"] = "rulebook"
    access_level_default: str | None = None
    enable_breadcrumbs: bool = True
    enable_contextual_summaries: bool = False
    cleaning: bool = True
```

Replaces scattered kwargs on `IngestionPipeline.run()`. Every future ingestion-time preprocessing option is added here.

### `KnowledgeChunk` (packages/rag/rag/knowledge/interface.py)

Two fields added (with defaults for backward compatibility):

```python
@dataclass
class KnowledgeChunk:
    chunk_id: str
    doc_id: str
    doc_title: str
    headline: str
    summary: str
    topic: str
    access_level: str
    scope: str
    text: str           # original_text from ChromaDB — now includes breadcrumb prefix
    rrf_score: float
    breadcrumb: str = ""           # NEW: structural location path
    source_type: str = "rulebook"  # NEW: document classification
```

### `EvalSummary` (packages/rag/rag/knowledge/evaluator.py)

New model `CategoryMetrics`; `EvalSummary` extended with an optional field:

```python
class CategoryMetrics(BaseModel):
    mean_mrr: float
    mean_ndcg: float
    mean_recall_at_k: float
    question_count: int

class EvalSummary(BaseModel):
    mean_mrr: float
    mean_ndcg: float
    mean_recall_at_k: float
    total_questions: int
    k: int
    category_scores: dict[str, CategoryMetrics] = {}  # NEW — empty = uncategorised run
```

### ChromaDB chunk metadata (pipeline.py `_build_records`)

Two fields added to the metadata dict written per chunk:

| Field | Type | Notes |
|-------|------|-------|
| `breadcrumb` | `str` | Structural path string; `""` when no heading found |
| `source_type` | `str` | Propagated from `IngestionPipeline.run()` parameter |

`original_text` field value changes: previously stored raw chunk text; now stores `"{breadcrumb}\n\n{raw_text}"` when breadcrumbs are enabled.

## State Transitions

### Re-ingestion requirement

Enabling breadcrumbs or contextual summaries on an **existing index** requires full re-ingestion (FR-014):
1. Clear the ChromaDB collection(s) for the affected documents
2. Re-run `IngestionPipeline.run()` with the new flags
3. The existing re-ingestion workflow (clear → re-ingest) is the supported path

Partial updates are not supported — compound text and `original_text` for existing chunks are immutable once stored.

### Benchmark results append-only

`benchmark_results.jsonl` is append-only. Previous records are never modified; `category_scores` is absent in old records and present in new records. Consumers must handle both shapes.
